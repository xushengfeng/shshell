尝试设计一种新的 shell 命令，用于方便输入，不需要处理流程逻辑，避免转义、歧义、多余输入

## 基本命令结构

`command arg1 arg2`

空格是识别基本结构的重要组成，如果参数带空格，可以通过`\ `转义

引号是最方便的区分参数的工具，只需要转义引号里面的引号即可。

引号自动匹配，空格附近的同类型引号才匹配。

引号前如果要打`\`而不作为转义符，需要使用`\\`。

除此之外引号不需要考虑其他任何东西。

不用引号包裹的 arg，存在以下转义符号：

`\`即使后面没有其他特殊字符，也要使用`\\`才得到单纯的`\`字符

任何地方都要处理：` ` `*` `?` `[` `]` 换行 `\`。

开头需要处理：`(` `'` `"` `$` `#`

结尾需要处理：`)` `'` `"` `;`

全部都是处理：`>` `>>` `2>` `&>` `|` `|2` `|&` `&&` `||`

其他转义如`\n`是用`echo -e`来代替

`#`表示注释

## 重定向

`>` 覆盖文件，`>>`追加文件

`>` stdout
`2>` stderr
`&>` both

## 管道

`|` stdout
`|2` stderr
`|&` both

## 其他命令输出作为输入的一个参数

`command (subcommand)`

不支持拓展为多参数

## 变量

`set x v`，相当于`set`命令的第一个参数是变量名，第二个是值，如果有多个，用空格组合在一起

使用`$变量名`引用，仅被空格区分时有用

变量作为 command 位置时报错，函数名称应该直接显示调用

## 函数

`function 函数名 (command1; command2)`或者

```
function 函数名 (
    command1
    command2
)
```

函数名可省略，使其为匿名函数，每个 command 其实就是函数，直接通过函数名即可调用，通过`$函数名`引用，或者在参数中通过`(函数名)`调用。

函数的返回使用最后一个命令的返回

## 别名

`alias a b`等同于`function a (b)`
